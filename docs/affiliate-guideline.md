# アフィリエイト運用ガイドライン

## 広告表記

広告・アフィリエイトリンクを含む可能性があるページには、「広告・PRを含む場合があります」と分かる形で表示します。

## ステマ対策

広告であることが読者に分かりにくい表示は避けます。SNS、記事、動画、LINEでも、広告やPRを含む場合は自然に分かる表記を入れます。

## Amazonアソシエイト表記

Amazonアソシエイトを利用する場合は、以下の表記を掲載します。

> Amazonのアソシエイトとして、AI副業キャンパスは適格販売により収入を得ています。

正式な審査・利用開始後は、最新の運営規約を確認します。

## 商品紹介の原則

- 利用中、実践中、調査中、紹介候補を分けて書く
- 公式サイトの確認を促す
- 向いていない人や注意点も書く
- ASP審査前はアフィリエイトURLを空欄にする
- リンクは `products.json` で管理し、本文に散らさない

## 断定表現を避ける

収益、成果、転職、案件獲得は個人差があります。「必ず」「誰でも」「簡単に」といった表現を避け、前提条件を明確にします。

## データ編集ルール

- 商品カードは `src/data/products.json` で管理する
- ASP表記は `src/data/affiliatePrograms.json` に集約する
- `isAffiliate: true` の商品は `disclosureRequired: true` を基本にする
- `programId` はASPデータの `id` と一致させる
- 広告URLが未設定の場合は、押せるボタンを出さず、カード上で「紹介準備中」または「PR掲載予定」と控えめに表示する
